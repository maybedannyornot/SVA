# SVA: SimpleVariantAnnotator
SVA is a simple Python tool for variant annotation. Based on a sequence (FASTA format) and its annotations (GFF format), it analyzes a list of variants (VCF format), determines the region in which they occur (intergenic/intron/exon), and identifies how they modify the protein sequence. This tool was created for a master's thesis titled "Analysis of ionizing radiation-induced genomic changes in _Monascus purpureus_ in the context of bioactive pigments biosynthesis".

## How to use?
1. Download "requirements.txt" file and install necessary dependencies.
2. Download .py file and move to your folder.
3. Run script using Python and enjoy.

For details type: python ./SimpleVariantAnnotator.py -h

## How it works?
The script builds a relational database (which it stores in RAM by default) based on the input GFF file. It then creates a list of genes sorted by start position within each read sequence to enable efficient searching. The next step involves removing introns and translating the sequence into an amino acid sequence. The script retrieves the coordinates of the coding regions for each gene from the database, extracts the sequences corresponding to the retrieved coordinates from a FASTA file, and combines them into a single continuous mRNA strand. If a gene is located on the complementary strand, the script reverses the extracted sequences to their complementary forms so that the sequence corresponds to the actual protein sequence. It then creates a dictionary that maps global positions of the entire sequence to local indices within the gene’s coding sequence and translates the mRNA into an amino acid sequence. Finally, the script sequentially processes the identified mutations loaded from a VCF file, starting by checking whether a given mutation occurs within a gene. If the mutation is intergenic, the script uses the bisection method to find the genes located upstream and downstream of the mutation site. It then calculates the distance and assigns a “+” or “-” sign to the calculated value, depending on whether the mutation is located upstream or downstream of the gene, taking into account which strand the gene is derived from. If the mutation is located within the gene, two cases are considered: when the mutation falls within a region that is skipped during splicing, and when the mutation occurs in the coding region. The script checks whether the created mapping dictionary contains the given global position. If not, this means that the searched-for position has no corresponding local index within the gene — a mutation in a non-coding region. In this situation, the script retrieves the coordinates of the coding regions of the gene from the database and iteratively checks which of them are closest to the analyzed mutation. It then calculates the distance from these regions and generates a result in a format analogous to that used for intergenic mutations. If, however, the global position has a local counterpart in the mapping dictionary, this means that the mutation occurs at a position that is part of the resulting protein. In this case, the script locates the mutation within the mRNA sequence and (based on its local index in the mapping dictionary) identifies the original codon where the mutation occurs. Next, the script verifies whether there are any other mutations in the analyzed codon; it then reconstructs the codon incorporating the mutations and translates both versions of the codon into amino acids. Based on this, the type of mutation is determined: synonymous or non-synonymous. 

The final detailed report contains information regarding the mutation site (sequence ID along with position) and the type of mutation site (intergenic, intron, exon), as well as the reference and alternative nucleotides. For mutations in coding regions, information is also provided regarding the impact of the mutation on the encoded protein, including the number of the modified amino acid, the codon sequence before and after the mutation, and their translation into amino acids. The tool also returns a summary of the number and types of mutations within each gene, as well as a file containing the protein sequences prior to the mutations. A VCF file is also generated, containing additional information from the detailed report on annotated variants.
